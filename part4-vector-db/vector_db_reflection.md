## Vector DB Use Case

A traditional keyword-based search would not be sufficient for searching large legal contracts. Such systems rely on exact keyword matches and fail to capture semantic meaning. For example, searching for "termination clauses" may miss relevant sections that use different wording like "agreement cancellation terms."

Vector databases solve this by converting text into embeddings that capture semantic meaning. This allows similarity-based search rather than exact keyword matching.

In this system, contracts would be converted into vector embeddings and stored in a vector database. When a lawyer asks a question in plain English, the query is also converted into an embedding. The system then retrieves the most semantically similar sections of the contract.

This enables accurate, context-aware search across large documents. It significantly improves usability and efficiency compared to traditional databases.

Hence, vector databases are essential for such use cases.
